# State Package

This package owns canonical Personal Model and State surfaces.

## Own Here

- Personal Model loading and file-backed baseline inputs while the reset is in
  progress
- identity-bearing elephant State projection
- relationship and durable personal-state projection
- prompt contract assembly for State-aware runtime surfaces
- canonical State persistence helpers and overlay logic

## Do Not Own Here

- tool `cwd` or `allowed_roots` instructions
- intent-era planning policy
- semantic index retrieval or memory ranking logic
- app-only rendering or transport behavior

State behavior should project one durable Personal Model and elephant State truth
across CLI, API, gateway, and dashboard surfaces.
